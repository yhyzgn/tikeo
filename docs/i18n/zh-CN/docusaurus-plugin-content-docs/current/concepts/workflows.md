# 工作流 DAG 模型

Tikeo 同时支持简单调度任务与工作流定义。工作流是由节点与边组成的 DAG，包含明确依赖、派发约束、重试行为和执行证据。

## 当前核心概念

- **Job**：可复用执行定义。
- **Instance**：一次 Job 或 Workflow 执行。
- **Attempt**：一次执行尝试。
- **Workflow DAG**：多步骤编排的节点与边。
- **Replay bundle**：事故复盘和可视化回放所需证据。

## UI 方向

Web 控制台包含工作流画布和拓扑视图。文档应先解释模型，再进入 JSON/YAML 细节。否则读者会把画布当成纯 UI 功能，而忽略运行时状态、依赖顺序、重试和审计。

## 安全规则

工作流文档必须区分已经实现的运行时行为和路线图能力。除非仓库里存在测试、fixture 或可运行 artifact，不要宣称某个可视化功能、迁移工具或自动修复能力已经完成。

## 评估清单

一次合格评估应证明的不只是“图能渲染”：创建或检查工作流定义，验证依赖顺序，触发运行，查看 instance/attempt 记录，并确认 replay 数据足以支持事故复盘。

## 失败处理模型

重试、取消、下游依赖行为必须显式。避免在公开文档中使用“自愈”这类模糊词，除非精确的 retry、backoff、rollback 或 compensation 行为已经实现并通过测试。

## Job 与 Workflow 的关系

Job 是可复用执行定义；Workflow 把 Job 和工作流本地节点组合成 DAG。拓扑和影响分析页面帮助操作者在修改 Job 或 Workflow 前理解依赖关系。

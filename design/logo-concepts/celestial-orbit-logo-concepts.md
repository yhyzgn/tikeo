# Tikeo 天体运动 Logo 概念板

预览文件：`design/logo-concepts/celestial-orbit-logo-concepts.html`

## 设计约束

- 不替换当前站点 Logo，只提供候选方向。
- 公转位置通过开普勒方程计算，不用 CSS 匀速圆周路径冒充天体运动。
- 天体自转独立于公转，保持恒定角速度。
- 呼吸律动只作用于 halo / 能量环，不改变轨道本身，避免破坏物理运动逻辑。
- 主色通过 CSS 变量跟随 `html[data-theme="dark|light"]` 切换，后续接入 Docusaurus 时可映射到 `html[data-theme]`。

## 候选方向

### 方案 A · 双星调度核

最接近现有 Logo 的精神延展：中心调度核 + 双天体围绕共同重心运动。适合作为主 Logo。

### 方案 B · 共振队列环

用 2:3:5 周期关系表达任务队列、重试和实例状态节律。中心恒星增加克制的小范围呼吸效果；三颗行星的可见轨道和 JS 运动参数同源生成，并为每颗行星增加一个本地卫星，避免“行星没有沿轨道跑”的错觉。

### 方案 C · 引力隧道门

倾斜轨道和中心门洞表达 Worker Tunnel / 出站控制平面。更偏企业基础设施感。

## 我的初步建议

优先推荐 **方案 A**。它保留当前 Logo 的“呼吸核心”识别，同时把运动逻辑升级为更可信的天体系统；在 favicon、导航栏、OG 图和启动页都比较容易缩放。方案 B 可作为动效扩展或 loading mark，方案 C 更适合 Worker Tunnel 子品牌/章节图标。
